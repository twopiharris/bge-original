Blender Game Engine Car Setup Checklist

Car body mesh

  • Build car body without wheels
  • rename car body
  • center of body near bottom center of frame

Car body physics

  • Game engine physics
  • physics type - rigid body
  • no sleeping - true
  • collision bounds - true
  • collision type - convex hull
  • mass - @400

Wheels

  • Wheel meshes NOT parented to body
  • placement doesn't matter
  • note radius for later use
  • physics type - no collision
  • name all wheels:
    • FLWheel
    • FRWheel
    • RLWheel
    • RRWheel

Car body game properties

  • turn (float 0.0)
  • speed (float 0.0)

Add scripts to text editor

  • CarSetup.py
  • Suspension.py
  • Powertrain.py

Logic State 1

  • Always sensor (no pulse mode)
  • always to and controller to state actuator
  • state actuator - go to state 2
  • set state 1 as initial state
  • always to python controller - CarSetup.py

Test carSetup

  • Add ground plane
  • Play game
  • Wheels should snap in place
  • car should fall to ground
  • In CarSetup.py, adjust tire positions
  • In CarSetup.py, adjust tire radius
  • Wait for other CarSetup adjustments

Logic State 2

  • Always sensor (no pulse mode)
  • always to and controller to state actuator
  • state actuator - go to state 3
  • always to python controller - Suspension.py

Logic State 3

  • Always sensor - pulse mode
  • Always to python controller - Powertrain.py
  • Add input sensors with pulse mode on:
    • Gas
    • Reverse
    • Brake
    • EBrake
    • Left
    • Right
  • All sensors to Powertrain.py

Test and adjust

  • Test turning, stopping, acceleration
  • Adjust various values to taste
  • power, speed limits, braking in Powertrain.py
  • tire position, radius, steering in CarSetup.py
  • Grip, suspensionStiffness, and RollInfluence in Suspension.py
